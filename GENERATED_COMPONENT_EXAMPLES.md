# Generated Component Examples

This document shows real examples of components generated by the fixed `generateReactComponent()` function.

## Example 1: Simple Button Component

### Input SemanticTreeNode:
```typescript
{
  tag: 'button',
  semanticName: 'Button',
  attributes: { class: 'btn-primary', style: 'padding: 16px' },
  children: [],
  text: 'Click me',
}
```

### Generated Output:
```typescript
import React from 'react';

interface ButtonProps {
  variant: "primary" | "secondary";
  label?: string;
}

export default function Button({ variant, label }: ButtonProps) {
  return (
    <button className="p-4">
Click me
</button>
  );
}
```

**Key Features:**
- ✓ Real button element (not a stub)
- ✓ Actual text content preserved
- ✓ Tailwind classes mapped from inline styles
- ✓ Props interface with inferred variant and label types
- ✓ Proper React component structure

**Key Features:**

## How It Works

### 1. Node-to-JSX Conversion
The `nodeToJSX()` function recursively walks the semantic tree:
- Extracts inline styles and maps to Tailwind classes
- Builds HTML attributes with proper React syntax
- Converts event handlers (onclick → onClick)
- Processes all children recursively
- Applies proper indentation

### 2. Props Inference
The `inferComponentProps()` function detects:
- **Variants**: From class names containing "primary", "secondary", "small", "large"
- **Labels**: From text content
- **URLs**: From href attributes
- **Data Attributes**: All data-* become typed props
- **Event Handlers**: From onclick, onchange attributes

### 3. Component Generation
The `generateReactComponent()` function:
- Calls nodeToJSX() for content
- Generates TypeScript interface from props
- Adds React import
- Returns complete, ready-to-use component
